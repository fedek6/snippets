# Optional chaining

<VersionTag version="es11" />

The optional chaining operator (?.) enables you to read the value of a property located deep within a chain of connected objects without having to check that each reference in the chain is valid.

If any chain link with `?` is `null` or `undefined` the whole expression will return `undefined`.

```js
const test = {
	a: {
		b: {
			c: "last"
		}
	}
}

console.log(test.a.b.c); // "last"
console.log(test.a.b.c?.d); // undefined
console.log(test.a.b.c?.d?.e); // undefined
```
